## 场景
现网中 IP 地址或网段可能存在冲突，需要登录多台网络设备采集信息进行分析。

## 架构设计

### 三层关系
```
角色定义（核心/汇聚/接入/出口/FW...）
    ↕ 设备被赋予角色
设备池（IP、厂商、认证）
    ↕ 场景按角色筛选设备
场景（如IP冲突 = 核心+汇聚+出口）
```

### 核心设计
- **角色**：设备的功能角色（核心/汇聚/接入/出口/防火墙），可灵活新增
- **设备**：每台设备可以有一个或多个角色标签
- **场景**：定义需要哪些角色参与，执行时自动匹配设备

### 场景可配置
- 场景定义存 JSON，不写死在代码里
- 每个场景可配置：所需角色、厂商命令、分析规则
- 新增场景只需加 JSON 文件，不改代码

## 环境
- 设备：华为 + 锐捷
- 堡垒机：纯跳板
- 运行：Windows 办公电脑 + GUI
- 数据：存本地 SQLite

## 采集引擎方案确定：paramiko

### 设备接入流程
- 堡垒机：域名登录，**SSH 密钥认证**
- 登录后 Expect 流程（4步跳到目标设备）：

  | 看到 | 发送 |
  |------|------|
  | `Opt or ID>:` | `n` |
  | `Opt or ID>:` | `0` |
  | `Opt or ID>:` | `=设备IP` |
  | `ID>:` | `0` |

- 跳到设备后，进入正常的 CLI 交互模式

### 优势
- 流转简单（4步固定），非常适合 paramiko 的 expect 模式
- 不需要 Xshell，运行时只依赖 Python + SSH（Windows/Linux 均可）
- 堡垒机登录流程可配置化：每个场景可定义 expect 规则列表

### 架构影响
- 采集引擎从「Xshell 驱动」改为「paramiko + expect 模式」
- 堡垒机 expect 规则可作为 JSON 配置的一部分
- 多设备可并行采集（paramiko 是纯 Python，线程安全）
- Xshell 依赖完全移除，工具可 Linux/Windows 双平台运行

## SSH 连接配置模型

### 连接独立可配
SSH 连接作为独立配置实体，每个连接包含：
- 名称（唯一标识，如 "深圳核心区堡垒机"）
- 主机（IP 或域名）
- 端口号
- 认证：SSH 私钥路径

### Section ↔ SSH 连接关联
- 设备4属性：Region（区域）、Section（分区）、Role（角色）、Vendor（厂商）
- Section 属性绑定 SSH 连接
- 同 Section 的设备，共享同一个 SSH 连接（同一台堡垒机入口）
- 不同 Section 的设备可能走不同的堡垒机

### 数据流示例
```
SSH连接A(Section: 核心A区)
├── 设备1 (IP: 10.0.1.1, Role: 核心)
└── 设备2 (IP: 10.0.2.1, Role: 汇聚)
     ↓ (通过同一堡垒机→跳不同设备IP)

SSH连接B(Section: 接入B区)
└── 设备3 (IP: 10.0.3.1, Role: 接入)
```

### 采集流程
1. 场景筛选出设备列表
2. 按 Section 分组，每组共享一个 SSH 连接
3. 每组设备用同一通道登录堡垒机，expect跳不同设备IP
4. 也可每组串行/并行采集

## SSH 连接配置补充：私钥密码

SSH 连接配置项补充：

| 字段 | 类型 | 说明 |
|------|------|------|
| 名称 | string | 唯一标识 |
| 主机 | string | IP 或域名 |
| 端口 | int | SSH 端口，默认 22 |
| 私钥路径 | string | SSH 私钥文件路径（如 `C:\Users\xxx\.ssh\id_rsa`） |
| 私钥密码 | string | 私钥的 passphrase（可选，无密码则为空） |

paramiko 加载带密码私钥的方式：
```python
from paramiko import RSAKey, SSHClient

# 无密码私钥
key = RSAKey.from_private_key_file(path/to/id_rsa)

# 有密码私钥
key = RSAKey.from_private_key_file(path/to/id_rsa, password=私钥密码)

# SSH连接
client = SSHClient()
client.set_missing_host_key_policy(AutoAddPolicy())
client.connect(hostname=堡垒机域名, port=22, username=user, pkey=key)
```

### 安全考虑
- 私钥密码在配置中建议加密存储（如本地 SQLite 加密字段）
- 运行时才解密，不落日志

## 全局配置

### 配置组成
工具首次使用时需完成全局配置：
1. SSH 连接配置（名称、主机、端口、用户名、私钥路径、私钥密码、expect/send流程）
2. Region+Section 到 SSH 连接的映射

### 映射规则
- Region + Section → SSH 连接
- 支持模糊匹配：`all` 表示该 Region 下所有 Section 映射到同一个连接
- 精确匹配优先级高于模糊匹配

示例：
- 北京四 + all → 大云堡垒机
- 芜湖202 + az1/az2/az3 → 苏州堡垒机
- 芜湖202 + pop → 大云堡垒机

### 导入导出
- 全局配置支持导出为 JSON
- 导出时私钥密码字段自动清空
- 支持从 JSON 文件导入
- 导入后需手动补填密码
